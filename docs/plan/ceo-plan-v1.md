## Vision

### 10x Check
A habit-breaking platform where shame is the social currency. Not celebrations, not badges, not points. The raw, honest, sometimes hilarious visibility of failure among friends. Hall of Shame > Hall of Fame.

### Core Insight
"抵抗" is not "didn't do it." It's "想做了，但我沒有做." Three states: unconscious, resisted, failed. This distinction makes every other habit tracker look lazy.

## Accepted Scope (V1)
- Pure client-side app (localStorage, no backend). DESIGN.md documents full V2 architecture; this plan is the simplified client-side subset.
- Next.js 14 App Router (client components)
- Dashboard with daily check-in (resisted / failed)
- Item CRUD (create, view, archive)
- Streak logic (resisted +1, failed = 0, no check-in = frozen). Day resets at midnight user's local timezone.
- All streak copy uses zero-based labels: Day 0, Day 1, Day n.
- Milestone celebrations (confetti at 7/14/30/60/100, using canvas-confetti)
- Heatmap calendar on item detail page (green=resisted, red=failed, gray=no check-in)
- PWA manifest (add to home screen, CapRover with Let's Encrypt SSL)
- Random snarky messages on each dashboard load (not "daily" — random per load from JSON array)
- Share streak screenshot (using html2canvas, generate image for download/share)
- Dark mode only, punitive aesthetic (dark bg, red accents, stark typography, shake animation on failure)
- Deploy on CapRover

### localStorage Schema
```typescript
interface NotToDoItem {
  id: string
  title: string
  description: string
  streak: number
  bestStreak: number
  lastCheckin: string | null  // ISO date
  isActive: boolean
  createdAt: string  // ISO timestamp
}

interface Checkin {
  id: string
  notToDoId: string
  date: string  // YYYY-MM-DD in user's local timezone
  status: 'resisted' | 'failed'
  createdAt: string
}

// localStorage keys
// 'ntd_items' -> JSON array of NotToDoItem
// 'ntd_checkins' -> JSON array of Checkin
// 'ntd_schema_version' -> number (for future migration)
```

## Deferred to V2
- Auth (NextAuth v5) + data sync from localStorage to PostgreSQL
- Drizzle + PostgreSQL (self-hosted on CapRover)
- Multi-user + challenges
- Hall of Shame (per-challenge view of who's on Day 0)
- Ranking (resistance ratio, streak, consciousness score)
- Confession on failure (optional text)
- Web Push notifications
- Max 3 active challenges per user

## NOT in scope
- AI-assisted habit coaching
- Public global leaderboard
- Paid features / monetization
- Mobile native app
